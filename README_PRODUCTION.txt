MATRIX PRODUCTION PIPELINE - OPERATOR MANUAL
================================================================================

This document explains how to use the automated production pipeline for the Matrix system.

1.  **Run Production Script**
    - Double-click `run_production.bat` to start the full pipeline.
    - This script will sequentially execute:
        1.  **Configuration Check:** Verifies API keys and database connection.
        2.  **Scout (Miner):** Harvests keywords (Long running process).
        3.  **Refiner:** Downloads PDFs and extracts data (Batch: 30).
        4.  **Composer:** Generates HTML articles (Batch: 10).
        5.  **Reporter:** Generates PDF audit reports (Batch: 10).

2.  **Customizing Batch Sizes**
    - You can edit `run_production.bat` to change the `--batch` numbers.
    - Recommended safe limits:
        - Refiner: 30-50 (Avoids memory issues)
        - Composer: 5-10 (Avoids API rate limits)
        - Reporter: 5-10 (Avoids PDF generation timeouts)

3.  **Troubleshooting**
    - If the script pauses with an error, check the console output.
    - Common errors:
        - **API Key Missing:** Check `matrix_config.py` or your environment variables.
        - **Supabase Connection Failed:** Check internet connection.
        - **Rate Limits:** Reduce batch sizes if you see 429 errors.

4.  **Stopping the Script**
    - You can press `Ctrl+C` at any time to stop the current python script.
    - The batch file is configured to pause and show an error message if ANY step fails or is interrupted abnormally.
